# Playwright E2E Testing

## Overview

Browser-level end-to-end tests that exercise the full application through the real UI — uploading HAR files, typing descriptions, verifying results, executing curl commands, and checking the response viewer. Powered by Playwright.

## Quick Start

```bash
# Install Playwright browser
npm run playwright:install

# Start both servers (required)
npm run dev

# Run all tests (mock + live)
npm run test:e2e

# Run with visual browser UI (watch tests execute in real time)
npm run test:e2e:ui

# Run in headed mode (see the actual browser window)
npx playwright test --headed
```

## Test Projects

| Project | Command | Tests | API Key Required |
|---------|---------|-------|-----------------|
| **mock** | `npm run test:e2e:mock` | 45 | No — backend is mocked via `page.route()` |
| **live** | `npm run test:e2e:live` | 4 | Yes — hits real backend + OpenAI pipeline |
| **all** | `npm run test:e2e` | 49 | Live tests need key, mock tests always work |

### What Each Test Covers

**smoke.spec.ts** — No backend needed. Verifies the page loads: header, hero text, upload zone, capability stats (71 domains, 5 languages, 8 filters), tips section, footer, nav buttons.

**upload.spec.ts** — No backend needed. Tests file input accepts `.har/.json`, displays filename + size after upload, shows HAR Inspector + Description sections, handles invalid JSON and missing `log.entries`, supports file replacement.

**analysis-mock.spec.ts** — Mocked backend via `page.route()`. Tests the full UI flow: upload → describe → analyze → pipeline stepper animation → result card with confidence badge → curl output → code generation tabs (Python, JS, Go, Ruby) → alternative matches → min 5 char validation → Cmd+Enter shortcut.

**analysis-live.spec.ts** — Real backend + OpenAI. Uploads actual captured HAR files (jokes-real.har, open-meteo-weather.har, pokeapi-pokemon.har, dog-ceo-random.har) through the browser UI, sends them to the real pipeline, and verifies the matched API appears in the result.

**proxy-execute.spec.ts** — Mocked analyze + proxy. Tests the Execute button, loading skeleton, Response card with status badge (200 OK), body tab with JSON, headers tab.

**collection.spec.ts** — Tests localStorage auto-save after analysis, History sidebar via H key and button click, sidebar close on Escape (checks `translate-x-full` CSS), Clear All button.

**keyboard.spec.ts** — Tests `?` (shortcuts modal), `Escape` (close), `H` (history), `I` (tech info), and verifies shortcuts are ignored when typing in the description textarea.

**har-inspector.spec.ts** — Tests table renders with correct request count badge, shows URLs and HTTP methods, example prompt buttons fill the description textarea.

**responsive.spec.ts** — Tests at 375x812 viewport: no horizontal scroll, upload zone usable, sidebar renders.

```bash
npm run capture:hars:extended # Capture 10 extended HAR files
npm run capture:all           # Run both capture scripts

```

## Configuration

**`playwright.config.ts`** at project root:
- `testDir: './e2e'`
- `workers: 1` (sequential — avoids backend overload)
- `timeout: 120_000` (2 min per test for LLM calls)
- Two projects: `mock` (excludes `*live*` files) and `live` (only `*live*` files)
- In CI: auto-starts backend + frontend via `webServer`
- Locally: expects servers already running (`npm run dev`)
- Screenshots on failure, trace on first retry, video retained on failure

## Architecture

```
Browser (Playwright)
    │
    ├── Mock tests: page.route() intercepts fetch to localhost:3001
    │   └── No real backend needed
    │
    └── Live tests: real fetch to backend
        └── Backend (localhost:3001) → OpenAI → curl result
            │
            └── Frontend (localhost:3000) displays result
                └── Playwright verifies UI elements
```
